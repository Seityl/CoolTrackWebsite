## Copywriting Guidelines

### Brand Voice & Tone

**Voice Attributes:**
- Professional but approachable
- Confident without being arrogant
- Technical yet accessible
- Helpful and educational

**Tone Variations:**

| Context | Tone | Example |
|---------|------|---------|
| Homepage Hero | Bold, Benefit-Driven | "Never lose another asset to temperature failure" |
| Features | Informative, Clear | "Monitor temperatures across unlimited locations from one dashboard" |
| Pricing | Transparent, Value-Focused | "Start protecting your assets for less than $1 per day" |
| Industry Pages | Empathetic, Expert | "We understand the challenges of maintaining pharmacy compliance" |
| Support/Legal | Professional, Clear | "Your data security and privacy are our top priorities" |

### Writing Principles

#### 1. Clarity Over Cleverness
```
❌ "Cool Track: Your temperature's best friend"
✅ "Monitor temperatures, receive alerts, maintain compliance"
```

#### 2. Benefits Before Features
```
❌ "24/7 cloud-based monitoring with real-time data sync"
✅ "Never miss a temperature alert - monitor your facility 24/7 from anywhere"
```

#### 3. Active Voice
```
❌ "Alerts can be customized by users"
✅ "Customize alerts for your specific needs"
```

#### 4. Specific Over Generic
```
❌ "Fast setup"
✅ "Setup in under 10 minutes"
```

#### 5. Remove Jargon
```
❌ "Leverage our IoT ecosystem for optimal environmental telemetry"
✅ "Connect sensors, monitor temperatures, protect your assets"
```

**Homepage Hero Headlines:**
1. [Benefit] + [Target Audience] + [Differentiator]

## Image Guidelines

### Image Requirements

#### Hero Images
- **Dimensions:** 1920x1080px (16:9 ratio)
- **Format:** WebP with JPG fallback
- **Size:** <200KB (optimized)
- **Content:** 
  - Product dashboard screenshot
  - Facility/warehouse with sensors
  - Professional monitoring environment
- **Text overlay:** Ensure 50% image area is solid color for text readability

#### Feature Images
- **Dimensions:** 800x600px (4:3 ratio)
- **Format:** WebP with PNG fallback
- **Size:** <150KB
- **Content:**
  - Product screenshots (dashboard, mobile app, alerts)
  - Annotated UI with callouts
  - Clean, professional, on-brand

#### Industry Images
- **Dimensions:** 1200x800px (3:2 ratio)
- **Format:** WebP with JPG fallback
- **Size:** <180KB
- **Content:**
  - Industry-specific environments
  - Professional, high-quality stock photos
  - Diverse representation
  - Authentic scenarios (no overly staged photos)

#### Icons
- **Format:** SVG (vector)
- **Style:** Line icons, consistent stroke width
- **Size:** 64x64px base size (scalable)
- **Colors:** Brand colors (primary blue, secondary orange)

#### Customer Logos
- **Format:** SVG preferred, PNG acceptable
- **Style:** Grayscale for consistency
- **Size:** Max width 200px, max height 80px
- **Spacing:** 40px between logos

### Image Optimization Checklist

- [ ] Resize to exact dimensions needed
- [ ] Compress (TinyPNG, ImageOptim, Squoosh)
- [ ] Convert to WebP format
- [ ] Include fallback format (JPG/PNG)
- [ ] Add descriptive alt text
- [ ] Implement lazy loading
- [ ] Use responsive images (srcset)
- [ ] File naming: descriptive-kebab-case.webp

### Alt Text Guidelines

**Formula:** [What it is] + [What it shows] + [Context if relevant]

**Examples:**
```
❌ "image1.jpg"
❌ "dashboard"
✅ "Cool Track temperature monitoring dashboard showing real-time sensor data"

❌ "warehouse"
✅ "Large warehouse with wireless temperature sensors installed on shelving units"

❌ "icon"
✅ "Alert icon representing instant temperature notifications"
```

**For decorative images:**
```html
<img src="pattern.webp" alt="" role="presentation">
```

### Image File Naming

**Convention:** `[type]-[description]-[variant].[format]`

**Examples:**
- `hero-homepage-dashboard.webp`
- `feature-alerts-mobile.webp`
- `industry-healthcare-pharmacy.jpg`
- `icon-realtime-monitoring.svg`
- `logo-customer-acme-corp.svg`
- `screenshot-dashboard-overview.webp`

## Meta Data Guide

### Title Tag Formula

**Homepage:**
```
[Product Name] - [Primary Keyword] | [Key Benefit]

Example: "Cool Track - Temperature Monitoring Software | Real-Time Alerts & Compliance"

Length: 50-60 characters
```

**Internal Pages:**
```
[Page Topic] | [Product Name]

Example: "Healthcare Temperature Monitoring | Cool Track"

Length: 50-60 characters
```

**Blog Posts:**
```
[Article Title] | [Product Name] Blog

Example: "FDA Compliance Guide for Pharmacies | Cool Track Blog"
```

### Meta Description Formula

**Homepage:**
```
[What it does] + [Key benefits] + [CTA or differentiator]

Example: "Cool Track delivers intelligent temperature monitoring with real-time alerts, comprehensive logging, and compliance reporting. Start your free 14-day trial today."

Length: 150-160 characters
```

**Features Page:**
```
[Discovery/Explore] + [Feature summary] + [Benefit]

Example: "Discover Cool Track's comprehensive temperature monitoring features including real-time alerts, cloud logging, mobile access, and automated compliance reporting."
```

**Industry Pages:**
```
[Industry-specific benefit] + [Solution] + [Compliance/outcome]

Example: "Ensure pharmaceutical compliance with Cool Track's hospital temperature monitoring. Real-time alerts, automated reporting, and 24/7 monitoring for healthcare facilities."
```

### Open Graph & Twitter Card

**Required Tags:**
```html
<!-- Open Graph -->
<meta property="og:title" content="[Page Title]">
<meta property="og:description" content="[Meta Description]">
<meta property="og:image" content="[1200x630px image URL]">
<meta property="og:url" content="[Page URL]">
<meta property="og:type" content="website">

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="[Page Title]">
<meta name="twitter:description" content="[Meta Description]">
<meta name="twitter:image" content="[1200x630px image URL]">
```

**Social Image Requirements:**
- Dimensions: 1200x630px
- Format: JPG or PNG
- Size: <300KB
- Content: Product logo + headline + visual element
- Text: Readable at small sizes

### URL Structure

**Best Practices:**
- Use hyphens, not underscores
- Lowercase only
- Short and descriptive
- Include target keyword when natural
- No unnecessary words (articles, conjunctions)

**Examples:**
```
✅ /features/
✅ /pricing/
✅ /industries/healthcare/
✅ /industries/food-beverage/
✅ /contact/

❌ /our-features/
❌ /pricing_page/
❌ /industries/healthcare-and-pharmaceutical-monitoring/
```

## SEO Checklist

### On-Page SEO

**Every Page Must Have:**
- [ ] Unique, keyword-optimized title tag (50-60 chars)
- [ ] Compelling meta description (150-160 chars)
- [ ] Single H1 tag with primary keyword
- [ ] Logical heading hierarchy (H1 > H2 > H3)
- [ ] Primary keyword in first 100 words
- [ ] Related keywords throughout content
- [ ] Internal links to related pages (3-5 minimum)
- [ ] Outbound links to authoritative sources (if applicable)
- [ ] Optimized images with alt text
- [ ] Structured data markup
- [ ] Mobile-friendly design
- [ ] Fast load time (<2 seconds)
- [ ] Clear CTA above the fold

### Technical SEO

**Site-Wide Requirements:**
- [ ] XML sitemap generated and submitted
- [ ] Robots.txt configured
- [ ] HTTPS enabled
- [ ] Canonical URLs set
- [ ] 404 page exists
- [ ] Page speed optimized (Lighthouse >90)
- [ ] Mobile responsive
- [ ] No broken links
- [ ] Proper redirects (301 for permanent)
- [ ] Breadcrumb navigation
- [ ] Schema markup implemented

### Content SEO

**Quality Indicators:**
- [ ] Original content (no plagiarism)
- [ ] Minimum word count met
- [ ] Proper grammar and spelling
- [ ] Easy to read (8th-grade level)
- [ ] Scannable (headings, bullets, short paragraphs)
- [ ] Provides value to reader
- [ ] Answers user questions
- [ ] Updated regularly
- [ ] Includes multimedia (images, videos)
- [ ] Encourages engagement (CTAs, forms)
